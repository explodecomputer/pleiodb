## Parent PRD

`issues/prd-pleiodbr.md`

## What to build

Test suite (`testthat`) with a small fixture `.pleiodb` dataset bundled in
`tests/fixtures/`. The fixture should cover:

- V = 100 variants (chr1, chr2, chr16, chr19)
- T = 20 traits (mix of trait IDs)
- A handful of deliberately imputed (v, t) pairs
- Pre-built `masks/5e-08.coo.zst` with a few significant hits
- A populated `rho.bin/.cidx`

Tests should cover:
- `open_pleiodb()` succeeds and returns correct V/T
- Format version guard fires on `format_version = 99`
- `phewas()` by ALID returns correct z for known pair
- `phewas()` by region returns correct variant count
- `gwas()` returns correct row count
- `tophits()` returns only rows below threshold
- `tophits()` errors when traits omitted
- `associations()` cross-product is correct
- `rho()` returns correct value for known pair
- `imputed` flag is TRUE for fixture imputed pairs and FALSE elsewhere

The fixture can be generated by a helper script
(`tests/fixtures/generate_fixture.py`) using the Python `pleiodb` package, but
the fixture itself is committed as binary so tests run without Python.

## Acceptance criteria

- [ ] `devtools::test()` passes with no failures
- [ ] Test coverage ≥ 80% of exported functions
- [ ] Fixture is committed to the repo and `R CMD check` passes
